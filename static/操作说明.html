微信 RSS 订阅系统 · 操作说明

从微信扫码授权开始，到用阅读器消费公众号文章 —— 一份可迭代的图文手册

目录

  1. 微信扫码授权（前置必做）
  2. 添加公众号订阅
  3. 抓取文章（自动 / 手动）
  4. 配置新文章通知（WebHook）
  5. 创建消息任务定时推送
  6. 生成 RSS 链接，在阅读器中订阅

1微信扫码授权（前置必做）

系统抓取公众号内容，依赖一个已登录的微信会话。所以第一步是完成授权。

  1. 登录系统后，进入左侧菜单 授权管理（/wechat-status）。
  2. 在 订阅管理 页（首页 /）右上角点击「刷新授权」，会弹出一张二维码。
  3. 打开手机微信，使用「扫一扫」扫描该二维码并确认登录。
  4. 回到 授权管理 页，点击「刷新状态」，当状态显示为「已授权」即成功。
授权会过期。过期后系统可通过你后续配置的通知渠道（见步骤 4、5）提醒你重新扫码。

2添加公众号订阅

授权完成后，就可以把想关注的公众号加进来。

  1. 在 订阅管理 页（/）左侧栏点击「订阅」下拉，选择「添加公众号」（/add-subscription）。
  2. 有两种添加方式：
    • 方式 A · 通过文章链接获取：粘贴一篇该公众号的文章链接，点击「获取」，系统自动识别公众号。
    • 方式 B · 搜索名称：直接输入公众号名称搜索并选择。
  3. 添加成功后，该公众号会出现在左侧列表中，点击即可查看其文章。
  4. 还支持「添加精选文章」「导出 / 导入公众号」「导出 OPML」等。

3抓取文章（自动 / 手动）

公众号加好后，文章会自动或手动同步进来。

  • 自动抓取：定时任务开启时（默认开启），系统按设定间隔自动轮询并更新文章。
  • 手动抓取：在订阅管理页选中某个公众号，点击右上角「刷新」立即抓取；「清理」可清除无效 / 重复 / 旧文章。

4配置新文章通知（WebHook）

想在有新文章时收到提醒，需要先在「配置信息」里填好通知渠道。

  1. 进入左侧菜单 配置信息（/configs）。
  2. 找到通知相关配置项，按需填写对应的 WebHook 地址：
    • notice.dingding —— 钉钉机器人
    • notice.wechat —— 企业微信 / 微信
    • notice.feishu —— 飞书机器人
    • notice.custom —— 自定义 WebHook
  3. 也可通过环境变量（如 DINGDING_WEBHOOK 等）配置，具体见项目 README。
只配置渠道还不够，还需在步骤 5 创建「消息任务」才会真正推送。

5创建消息任务定时推送

消息任务负责把新文章按设定时间推送到上一步配置的渠道。

  1. 进入左侧菜单 消息任务（/message-tasks）。
  2. 点击「添加任务」：选择推送类型（如 WebHook / 浏览器通知等），并设置 cron 定时规则。
  3. 保存后，任务会按 cron 定时把新文章推送到对应渠道。
  4. 同页面也可开启浏览器通知：开启后网页端会弹出新文章提醒（需浏览器授权通知权限）。

6生成 RSS 链接，在阅读器中订阅

把公众号内容变成标准订阅源，用你习惯的阅读器消费。

  1. 在订阅管理页选中某个公众号，点击右上角「订阅」下拉。
  2. 选择需要的格式链接：RSS / ATOM / JSON / Markdown / Text。
  3. 复制生成的链接（形如 /feed/<公众号ID>.rss），添加到任意 RSS 阅读器（推荐 folo）即可像普通订阅源一样消费。
最小上手路径：启动后端 → 登录 → 授权管理扫码 → 添加公众号 → 等几分钟或点刷新 → 在「消息任务」配一个 WebHook 通知 → 把 RSS 链接丢进阅读器。
