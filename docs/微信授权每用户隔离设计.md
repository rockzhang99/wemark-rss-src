# 微信授权每用户隔离设计

## 背景

原系统微信授权为**全局单 token**：扫码登录后 token/cookie 仅存一份（`data/wx.lic` 或 Redis `werss:token:data`），所有用户共享同一微信登录态。任何登录用户在前端"公众号状态"页都能看到这同一个全局授权，互不隔离。

本次改造目标：**微信授权从全局单 token 改为每用户各自授权**；用户之间看不到其他人的授权；管理员也只能管理自己的 AK（AK 部分本就按 `user_id` 隔离，已满足）。

## 设计

### 1. 存储：新增 `user_wechat_auth` 表

| 字段 | 说明 |
|---|---|
| `user_id` (PK) | 用户名，即授权归属 |
| `token` | 微信 token |
| `cookie` | 微信 cookie 字符串 |
| `fingerprint` | 浏览器指纹 |
| `expiry` | JSON：过期信息 |
| `ext_data` | JSON：公众号信息（名称/头像/粉丝数等） |
| `login` | 是否已授权 |
| `updated_at` | 更新时间 |

新表由 `Base.metadata.create_all` 在启动时自动建表，无需 ALTER 迁移。

### 2. 授权写入：扫码/切换归属当前用户

- `apis/auth.py` 的 `/qr/code`、`/switch` 在调用前设置 `WX_API._auth_user = current_user["username"]`；
- `driver/wx.py` 的 `Call_Success`（扫码成功回调）在写 token 时，若 `_auth_user` 存在，则额外调用 `set_token_for_user(user_id, ...)` 写入该用户表；
- 同时保留全局 `set_token`（作为未授权用户的回退，确保抓取不中断）；
- `/qr/over` 后清除 `_auth_user` 上下文。

### 3. 授权读取：按用户，回退全局

`driver/token.py` 新增：
- `set_token_for_user(user_id, data, ext_data)`：写用户表；
- `get_user_wx_session(user_id)`：读用户表；
- `get_active_wx_session(prefer_user_id)`：**优先用户，回退全局**（兼容旧数据/未授权用户）。

### 4. 抓取链路：按任务归属用户取 token

- `core/wx/base.py` 的 `WxGather.__init__(user_id=None)` / `get_token(user_id)` 透传 `user_id`；
- `Model()` 创建子类（`MpsWeb`/`MpsAppMsg`/`MpsApi`）时把 `user_id` 传给子类实例，确保抓取用对应用户 token；
- `jobs/mps.py` 的 `do_job`：`WxGather(user_id=task.owner).Model()`，定时/手动任务按任务 owner 取授权；
- `apis/mps.py` 的 `update_mps` 单公众号同步：`WxGather(user_id=current_user["username"]).Model()`；
- `fetch_all_article()`（旧全量同步）无 owner，回退全局。

### 5. 状态展示：按当前用户

- 新增 `GET /auth/wx/status`：返回当前用户自己的授权状态（login/token/expiry_time/info），互不可见；
- `apis/sys_info.py` 的 `/sys/info` 的 `wx` 部分改为按 `current_user["username"]` 返回（回退全局）；
- 前端 `WechatStatus.vue` 改为调用 `/auth/wx/status`，展示当前用户自己的微信授权。

## 行为变化

| 场景 | 改造前 | 改造后 |
|---|---|---|
| 用户 A 扫码授权 | 写全局，所有人可见 | 仅写入 A 的用户表，仅 A 可见 |
| 用户 B 看授权状态 | 看到 A 的全局授权 | 看到自己的（未授权则回退全局/空） |
| 定时任务抓取 | 用全局 token | 用任务 owner 的 token，无则回退全局 |
| 管理员 AK 管理 | 只看自己的 AK | 不变（本就只看自己的） |

## 兼容性

- 全局 token 仍保留作为**回退**：未授权用户、旧数据、全量同步 `fetch_all_article` 仍可用全局 token 抓取，不破坏现有功能；
- 存量数据：旧全局 token 仍在 `data/wx.lic`/Redis，未授权用户自动回退使用。

## 改动文件

| 文件 | 改动 |
|---|---|
| `core/models/wechat_auth.py`（新） | 用户微信授权模型 |
| `core/models/__init__.py` | 注册 `UserWechatAuth` |
| `driver/token.py` | 新增用户维度存储/读取 + `get_active_wx_session` 回退 |
| `core/wx/base.py` | `WxGather` 透传 `user_id`，按用户取 token |

| `web_ui/src/views/WechatStatus.vue` | 调 `/auth/wx/status` 展示当前用户授权 |

## 验证

- 后端 `py_compile` 全部通过，无 lint 错误；
- 前端 `npm run build` 通过；
- 建议启动后：用用户 A 扫码授权 → 用户 B 登录看不到 A 的授权；A 的订阅抓取用 A 的 token；未授权用户回退全局 token。
